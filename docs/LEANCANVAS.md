## 상세 설명

### 1. 문제 (Problem)

소모임 주최자가 반복적으로 겪는 세 가지 핵심 문제:

1. **공지의 휘발성**: 채팅방 공지는 새 메시지에 묻히고, 참여자들이 확인을 못 하는 경우가 많음
2. **수동 취합의 비효율**: "가요", "못 가요" 댓글을 일일이 세거나, 구글 폼 응답을 복사해서 정리해야 함
3. **출석 체크의 번거로움**: 당일 현장에서 명단을 꺼내 하나씩 체크해야 하고, 디지털화가 안 되어 있음

**기존 대안의 한계**: 카카오톡 + 구글 폼 + 스프레드시트 조합은 3개 도구를 오가야 하고, 참여자도 여러 링크를 클릭해야 하는 불편함이 있음

---

### 2. 고객 세그먼트 (Customer Segments)

| 세그먼트            | 특징                                                                    |
| ------------------- | ----------------------------------------------------------------------- |
| **주최자 (핵심)**   | 정기/비정기 소모임 운영자. 수영, 헬스, 독서, 친목, 스터디 등            |
| **참여자 (비회원)** | 링크를 받아 접근하는 일반인. 별도 가입 없이 이름만으로 신청             |
| **회원 참여자**     | 여러 모임에 참여하며 본인 계정으로 신청 이력 관리를 원하는 사용자       |
| **얼리어답터**      | 카카오톡 공지 방식에 불편함을 느끼고 디지털 도구 도입에 적극적인 주최자 |

---

### 3. 가치 제안 (Unique Value Proposition)

**고유 가치 제안**: 링크 하나로 공지·신청·출석을 모두 해결

**차별화 포인트**:

- 참여자가 회원가입 없이 이름만으로 즉시 신청 가능 → 링크 공유만으로 즉각적인 참여 수집
- 주최자 입장에서 하나의 화면에서 모임 생성부터 출석 체크까지 완결

---

### 4. 솔루션 (Solution)

| 문제      | 솔루션                                                                |
| --------- | --------------------------------------------------------------------- |
| 공지 휘발 | 슬러그 기반 고정 URL(`/events/{slug}`)로 항상 접근 가능한 모임 페이지 |
| 수동 취합 | 참여 신청 폼 + 실시간 참여자 목록 → 주최자가 확정/취소 처리           |
| 출석 체크 | 전용 출석 체크 화면에서 체크박스로 간편 처리 + 현황 요약(n/전체)      |

---

### 5. 채널 (Channels)

1. **바이럴 채널**: 주최자가 카카오톡에 링크 공유 → 참여자들이 자연스럽게 서비스 경험
2. **커뮤니티 마케팅**: 동호회 카페, 맘카페, 클래스101 등 소모임 커뮤니티 홍보
3. **입소문**: 참여자가 다음 모임 주최 시 서비스 재사용 (주최자 전환)

### 9. 경쟁우위 (Unfair Advantage)

1. **비회원 참여 신청**: 참여자에게 가입 장벽 제로 → 링크 공유만으로 즉각적인 참여 수집 가능. 경쟁 서비스 대부분 회원가입 필요
2. **모임 유형 무관**: 수영, 헬스, 독서, 친목, 스터디 등 모든 소모임에 적용 가능한 범용성
3. **단순함**: 주최자 입장에서 배울 것이 없는 직관적인 UX. 링크 생성 → 공유 → 관리의 3단계
4. **바이럴 구조**: 참여자가 주최자로 전환되는 자연스러운 성장 루프
